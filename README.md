# Interaksjonsdesign tema 8: Applikasjonsutvikling
This repo supports the <b>Application development</b> course at Interaction Design, Høyskolen Kristiania, spring 2020. Subjects and excercises are ongoingly submitted from the teaching staff. 

## How to use
As a student, first <b>clone</b> the repo to your local machine. The easiest way to do this, is to use the green button on the right hand side of the repo frontpage, and choose 'open in desktop' (you need <a href='https://desktop.github.com/'>github desktop</a> installed). You now have a personal copy of the repo on your local machine. 

At any point you can choose to <b>pull</b> the repo (in github desktop it's called 'fetch origin') from the remote, to get the files. If you accidentally change or remove files from the repo and wish to get back to the original version, just delete your local copy and clone the repo again.  

The repo is <b>not</b> intended as scaffolding for personal work - since you are not able to push your own changes to it. It's basically just a bunch of examples from the online classes. If you wish to use the examples as a starting point for your own work, just copy any project into a personal folder.

## Overordnet fokus

I dette temaet skal studentene fordype tekniske ferdigheter innenfor front-end utvikling. I tillegg skal studentene (videre)utvikle evnen til å selvstendig hente inn og ta i bruk relevant kunnskap.


1. Programmering og algoritmisk tenkning
2. System APIs
3. Versjonshåndtering (GIT)

I dette tema skal vi for det første trene måten vi går til forskjellige programmerings oppgaver. Algoritmisk tenkning går kort sagt ut på å dele et problem opp i mindre biter, så man får bedre oversikt over hva som må gjøres og i hvilken rekkefølge. Algoritmisk tenkning - eller med et bredere uttrykk - [computational thinking](https://www.bbc.co.uk/bitesize/guides/zp92mp3/revision/1) - er *ikke* en kode disiplin, men en måte å tilgå problemløsning i en digitalisert verden. Dette skal trenes i dette tema. 

I [temaoppgaven](https://paper.dropbox.com/doc/Temaoppgave--AxrRpuTbuMc7nOAoDkhNk4bgAQ-S6TxCNYtXbOCAyULUApZt) skal det utvikles en såkaldt «nytte applikasjon», en app som kan hjelpe med stort eller smått i hverdagen til en bruker. Ettersom funksjonaliteten til app’en er frivillig, får vi rik mulighet til å se på forskjellige strategier til å løse en sådan oppgave ved å dele den opp i mindre biter. 

Vi får også leilighet til å se nærmere på begrepet «reaktivitet», hvordan digitale tjenester i stigende grad blir noe som virker tilbake på virkeligheten i realtid. Både interaktivt, ved å reagere på bruker input, og ved å kople sig opp på live datatjenester mm. Dette fokus understøttes av rammeverket svelte.js som dere allerede har litt erfaring med, og som gør prototyping ganske kjapp og effektiv.   

Den app dere skal utvikle blir gjennom verktøyet [electron](https://www.electronjs.org/) til en desktop applikasjon. På tross av at vi utvikler den med vanlig nett teknologi - html, css og javascript - ender den altså opp som en app på skrivebordet som dem du henter i app-store eller .exe filer. Dermed får vi også mulighet for å gå i dybden med temaets annet fokus: system API’s. Et API er grensesnitt, ikke (som vanlig) for en bruker, men for kode. Electron sitt API gir os mulighet for å bygge en nettside som kan kompileres til en desktop applikasjon.      

Når vi er ved API’er har jeg også lyst til å introdusere jer for en annen form av disse: data API’er. Kort sagt informasjons ressurser på nettet som er gjort tilgjengelige via et grensesnitt. Vi kommer ikke til å gå i dybden med dette endda, men dere får sett et par eksempler på hvordan man henter data og bruker dem i sin egen kontekst (nettside). Dette kan tjene som inspirasjon eller brukes direkte i temaoppgavene. 

Til slut må vi også ha en grundig introduksjon til versjonshåndtering med Github i dette temaet. Dere har allerede brukt Github til forskjellige prosjekter, men det er fortsatt mye og skjønne og venne sig til.
